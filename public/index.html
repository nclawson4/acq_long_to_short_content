How I built it
Validation

Accuracy is enforced in code, not graded after the fact.

Before any clip ships, it goes through a QC gate of eight checks against the finished mp4. The gate's truth sources are ffprobe and ffmpeg ebur128 reading the file directly, not the tools that produced it. A single failure drops the clip. After the run, three more checks grade the whole job before it's marked done.

Underneath the QC gate, every stage runs inside a try / retry loop in the orchestrator with budget enforcement from the cost ledger. Exceptions are caught, retried up to twice, and translated into named alarms with one of six recommended actions (continue, retry_stage, retry_with_adjusted_args, drop_clip, escalate_human, abort_run). The orchestrator picks the action; the QC gate decides whether the result is shippable.

Failure recovery

When something breaks, you find out from the system, not from a user.

A pipeline that nobody can see is a pipeline that doesn't work. Three layers (typed alarms inside the run, an external webhook on critical failures, and a scheduled heartbeat from the edge) make sure that every kind of failure leaves a paged human, not a stuck spinner.

1

Typed alarms inside the run

Every guardrail, checkpoint, and tool fires named alarms with a severity (info / warning / error / critical) and a recommended action (retry_stage, drop_clip, escalate_human, abort_run). The orchestrator reads them and decides. 19 alarm names are pre-registered so no stage can invent its own.

2

Webhook on error + critical

Every error or critical alarm posts to ACQ_ALARM_WEBHOOK_URL in the background. Slack, Discord, or PagerDuty Events v2 are auto-detected by URL. Payload includes alarm name, stage, recommended action, and context. No webhook configured = silent no-op; pipeline never blocks on the call.

3

Heartbeat from Vercel cron

/api/heartbeat runs every 5 min from the edge, probes the tunnel + runner, and fires the same webhook if anything's wrong: tunnel unreachable, secret unset, or blob token missing. Catches outages the per-run alarms can't see because no one is running anything.

Cost optimization

Pennies per clip vs. an editor's hour.

Real measurements from the 28-video gold set: the pipeline averages $0.054 per finished short. A freelance editor producing the equivalent short from a long-form interview charges between $40 and $150 depending on turnaround. The economics only work at volume, and we say so below, but at volume the gap is roughly 1,000× to 3,000× on direct unit cost.

Freelance editor $40–150 per finished short
  • 1–2 hours labor to pick the moment, crop, caption, polish
  • Variable quality, depending on the editor's eye for hooks
  • Turnaround: next-day at best for one-off; days to weeks at volume
  • Scaling: linear; every extra clip is another billable hour
This pipeline (measured) $0.054 per finished short, end-to-end
  • ~131s wall time per video, fully unattended
  • Deterministic QC gate; clips that fail the bar never ship
  • Turnaround: seconds from "paste URL" to "download MP4"
  • Scaling: sub-linear; same code, more compute when needed
